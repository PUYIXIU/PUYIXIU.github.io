插件开发

有很多大名鼎鼎的插件，比如FeHelper、Convertio等等。
平时在用浏览器插件时，也会好奇这些插件到底是如何开发的。
大多数主流浏览器都提供了很好的插件开发环境，
插件项目的结构与我们平时开发web项目也大同小异，但是需要注意的是manifest.json文件，
manifest.json配置文件用于像浏览器介绍这款脚本的一些基本信息。

某天我正在开发一个基于Flask的塔罗牌算命网站，
需要给每张卡牌提供一张图片，于是就开始网上冲浪寻找图片资源，
但是塔罗牌（大阿卡纳牌）有足足22张！
难不成我要重复右键→图片另存为→保存整整22次吗？
达咩，我这该死的叛逆心被成功激起，
于是就想干脆趁这个机会涉足插件开发的领域。

资源抓取插件

当然我这个前端三脚猫一时兴起开发的小玩意没法跟很多应用商店里比较成熟的插件相比，
目前已经实现网站资源抓取的优秀插件也有很多：

  • Image Downloader

简单看了Image Downloader的源码，也比较简单，打算整理完自己开发的经验后好好看看别人的代码。

思路介绍

项目结构
  • static/js/getImage.js 在ContentScripts中运行的脚本
  • manifest.json 插件配置文件
  • popup.html 插件调用时弹窗页面
  • popup.js 弹窗页面对应脚本
  • manifest.json
  • getImage.js
  • icons 不同分辨率下的icon图片
  • name 插件名称
  • version 版本
  • description 描述
  • manifest_version 使用menifest的版本
  • content_scripts 对导入content_script中的脚本的配置
    • all_frames 是否在页面内所有content_scripts中都执行此脚本
    • matches 匹配到下列路径时执行
    • js 执行包括的js文件
    • run_at 指定注入脚本的执行时机，可以是如下值：
      • document_start 文档开始加载时
      • document_end 文档已完全加载
      • document_idle 文档完完全加载并空闲
      • document_end 整个DOM树加载完成

getImage.js是要注入content-script的主运行脚本，
此外还引用了jquery库，用来做dom操作和ajax请求。

页面中暴露了一个黄色按钮，用于触发getImage中绑定的读取图片事件,
触发按钮点击事件之后，页面滚动条自动下沉直到页尾（这一步是为了将做了图片资源懒加载的页面图片加载出来），
获取页面image标签的src地址，在弹窗中展示图片信息，点击图片后通过src请求图片资源。

开发时碰到的问题

因为PicBoo比较简陋，甚至可以说相当毛坯，
光是在开发的时候我就发现了各种各种未完待续的bug，比如：

  • 页面懒加载导致图片资源加载不全
  • 只通过image的src标签获取图片资源，其他方式加载的图片统统获取不到（如css的background-image）

但是STOP！！！
PicBoo毕竟只是我对浏览器插件一次试探性的成果，光是在获取网页图片这一步上我就花费了不少功夫（菜）。

关于跨域请求资源

因为不了解浏览器关于跨域请求资源的限制，我被卡了很久…很久…真的很久。
事情是这样的：
我查了些资料，知道了浏览器插件都有一个popup.html窗口文件，于是就开始在popup.html上搞装修，
和所有缺少想象力的平庸码农一样，我尝试用百度的logo做图片下载测试，
创建了一个a标签，给它加了href和download属性，并开始触发它：
《图片》
完美失败，网站没有下载图片，莫名其妙跳转到图片页面，
这就很奇怪，因为明明只要给a标签配置download属性就能表示下载，为什么还是会跳转？
我百思不得其解，卡了一段时间后我想：不能下载，我只让它显示总行了吧？
于是放弃使用a标签，开始动态创建image，结果：
完美失败*2，连图片资源都申请不到，更加迷惑。
再看报错，似乎是说不能跨域请求资源，烦死。
但是明明在html里直接写image标签，配置跨域src时就能请求到，怎么到了js这就区别对待？这谁规定的？
一查资料，哦，原来是浏览器规定的，那没事了。

脚本文件的加载方式会受到浏览器的安全机制限制
浏览器同源策略：
协议、域名、端口号都必须相同

怎么解决跨域呢？我问ChatGPT:

如何使用纯前端的方式解决跨域？

GPT回答的那叫一个干净利落：”不存在纯前端的解决方式。“
好嘛，我可不要再开一个后台专门为了请求跨域资源，
跨域，跨域，烦死，干脆让这网页自己请求自己总不算跨域了吧？
果真有这个方式，那就是Content—Script，运行在网页内部，
于是只好在Content-Script里面请求图片资源，
好消息是请求成功啦！坏消息是下载不下来，
我动态创建的a标签p用没有，烦死，我问ChatGPT这特么怎么回事，
它回了一段代码，我一看这不跟我写得一模一样吗。

不行啊！这么写没用啊！

我的精神状态很接近赛博精神病，开始对着GPT发羊癫疯，
神奇的事情发生了，GPT竟然又回我了一段代码，说实话跟之前的差别不大：

但事情就这样神奇的得到了解决，oh，how strange.
